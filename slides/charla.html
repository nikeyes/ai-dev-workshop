The 4 problems

What AI does poorly

01
No context

At the start of every session, the AI knows nothing about your codebase. Zero.

02
Hallucinations

It invents APIs, methods, and dependencies that don't exist — and sounds completely sure.

03
Non-determinism

The same prompt can produce different results every time you run it.

04
Hard to verify

We generate code faster than our cognitive load allows us to review it.

Advanced testing

Mutation testing, Acceptance Testing, Architectural Testing — all expand the feedback loop.

The feedback loop is what ensures we're doing the right things — and doing them right. With AI generating more code than ever, resist the giant PR.

07
Same principles, new scale

TDD hasn't died. It changed scale.

What expires: the ritual

Write a tiny test → minimal implementation → refactor → repeat.
Micro-cycles add steps and tokens the agent doesn't need.

What endures: the idea

Define expected behavior BEFORE implementing.
With AI this is more important, not less: the test is your verification contract.

The new cycle
  • Define acceptance tests at a functional level
  • Let the agent implement everything at once
  • Run the tests
  • Iterate if they fail

Test-first, but at macro granularity. Not micro-steps — verifiable macro-objectives.

08
Design for oversight

You can't review every line.
Design systems that catch mistakes.

Automated

Technical guardrails

  • Hooks: automatic linter on every tool use
  • Architecture tests: verify structure, not just functionality
  • Acceptance Testing: expected behavior defined upfront
  • Mature CI/CD: if it's not reliable with humans, it'll be chaos with agents
  • Agent permissions: define what it can do without approval — and what it can't
Human judgement

Pre-merge checklist

  • Does this respect my domain or just sound right?
  • Do the tests actually verify something real?
  • Is there observability for this use case?
  • Could I delete this code without breaking anything?

Guardrails catch the obvious. The checklist catches what only you can see.

09
Agent security

What can the agent do without your permission?

AI generates the code. You ship it. You own it.
You are responsible for every line that reaches production.

Without approval
  • Edit code
  • Run tests
  • Generate docs
  • Create branches
  • Open draft PRs
vs
Always with human approval
  • Push to main
  • Modify CI/CD pipelines
  • Touch IAM / Terraform / infra
  • Access secrets or credentials
  • Self-approve its own PRs
Source: Veracode — GenAI Code Security Report 2025

The transformation

What changes?

What you do

Your role

  • ✕ You're no longer paid for writing code
  • ✕ Nor for using AI
  • ✓ You're paid for deciding what should NOT go to production
  • ✓ You're paid for protecting the system
What surrounds you

The environment

  • → The code-writing tool
  • → The speed of generation
  • → The economic barrier to rigorous testing
  • → The format of documentation (CLAUDE.md, skills)
  • → Who writes the code (human vs. agent)

The tools change. The responsibility doesn't.

What matters

What endures — and grows stronger?

01
Simple architecture and design

AI generates pieces. The challenge is integrating and maintaining them over time.

02
Small safe steps

The bottleneck was never generating code — it was shortening the feedback loop.

03
Define behavior before implementing

If you don't define it, AI will write perfect tests for the code it decided on itself.

04
Autonomy calibrated to risk

Not everything needs the same level of rigor. But every level needs its guardrails.

05
Teamwork and clear communication

AI amplifies what's already there. If we're misaligned, it will amplify the chaos.

13
Summary

5 key takeaways

1

AI is a powerful tool, but without discipline it slows you down or creates more problems than it solves.

2

Your competitive advantage is no longer writing code — it's understanding the problem.

3

What AI generates by default is a draft that needs configuration.

4

Research → Plan → Implement → Validate. Don't do everything at once.

5

Adopting AI requires training, community, and metrics — not just licences.
