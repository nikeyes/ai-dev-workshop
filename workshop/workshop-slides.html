Bloque 1 · Paso 3

Memoria de los agentes

Sin CLAUDE.md

El agente empieza cada sesión desde cero. Es como contratar a alguien nuevo cada día.

Con CLAUDE.md

El agente tiene contexto persistente: convenciones del proyecto, cómo validar cambios, qué evitar. No tienes que repetirlo nunca más.

Con Rules

Reglas específicas que el agente aplica siempre, organizadas por módulo o tipo de tarea.

Bloque 1 · Paso 3 · Buenas prácticas

¿Qué poner en el CLAUDE.md?

  • Escríbelo con tus propias palabras, como si se lo explicaras a un compañero nuevo para que no cometa errores.
  • Mantenlo corto (150–300 líneas máx).
  • Solo cosas que no se pueden deducir del código.
  • No dupliques lo que ya está en el README, docs o configs estándar.
  • Define convenciones del proyecto que difieran del estándar del lenguaje.
  • Indica cómo validar los cambios (tests, linter, build).
Bloque 1 · Paso 3 · Jerarquía

Dónde está el CLAUDE.md

Ubicación Alcance
~/.claude/CLAUDE.md Todos tus proyectos — convenciones personales
<raíz>/CLAUDE.md El proyecto completo
<subdirectorio>/CLAUDE.md Solo ese módulo — útil en monorepos
.claude/rules/ Reglas opcionales organizadas por fichero
Prioridad

Cuanto más cerca del código, más específico. El subdirectorio sobreescribe al proyecto, que sobreescribe al global.

Bloque 1 · Paso 4

Contexto

> /context > /compact
/context

Muestra cuánto contexto llevas consumido en la sesión actual. Puedes activarlo permanentemente en la status bar.

⚠️ /compact — con cuidado

Claude lo ejecuta solo cuando no queda espacio. Puede perder contexto crítico que no has guardado. Si el agente "olvida" algo de repente, puede ser esto.

Bloque 1 · Paso 4

Añadir contexto de otro módulo

> /add-dir ../otro-servicio
Cuándo usarlo

Cuando necesitas que el agente entienda código de otro módulo sin abrir una sesión nueva. Útil en monorepos o cuando un cambio cruza servicios.

Alternativa

Si el otro módulo tiene su propio CLAUDE.md, Claude lo leerá automáticamente al añadir el directorio.

Bloque 1 · Paso 7

Hooks — guardarraíles automáticos

Qué son

Comandos que se ejecutan automáticamente en respuesta a acciones del agente. El agente no los pide — simplemente ocurren.

Evento Se dispara cuando… Ideal para
PostToolUse + Edit Después de cada edición de fichero Linter / formatter — actúa mientras el fichero está en foco
Stop Cuando el agente termina su turno Build / tests — espera a que todos los cambios estén hechos

Bloque 1 · Paso 8

Agents & Skills

El problema

Cada persona de tu equipo reinventa los mismos prompts. Las Skills son la forma de compartir conocimiento entre sesiones y entre personas.

Qué es una Skill

Un fichero de instrucciones que el agente carga cuando lo necesita. Vive en .claude/skills/ y se activa con /nombre.

Agentes

Claude puede delegar subtareas a subagentes especializados. Cada agente tiene su propio contexto, herramientas y CLAUDE.md.

Bloque 2 — 1h

Tu ticket, tu repo

Bloque 2 · 1h

Tu ticket, tu repo

  • Cada persona trabaja su propio ticket.
  • El objetivo no es terminar el ticket entero.
  • El objetivo es aprender a usar y configurar Claude Code con código real vuestro.
  • Al principio iréis inseguros y lentos. Cuando tengáis el repo configurado, iréis volando.
Bloque 2 · Guía

Mientras trabajas

  • Configura los permisos que necesitas para tu ticket de forma iterativa.
  • Si Claude Code no se comporta como esperas, añádelo al CLAUDE.md o a las Rules.
  • Si escribes el mismo prompt más de 2 veces: sácalo a una Skill.
  • Fíjate en cómo se va llenando el contexto.
  • Añade hooks para el linter, formatter u otras validaciones.
